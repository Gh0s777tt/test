## Technical Specifications

### ARM64 Memory Layout

```
0x40000000: Kernel code (.text)
0x40080000: Kernel data (.data, .rodata)
0x40100000: Kernel BSS (.bss)
0x40180000: Stack (8MB)
0x40980000: Heap (16MB)
0x41980000: Page tables
0x41A00000: Device memory
```

### ARM64 Page Table Entry

**Structure**:
- Bits 0-0: Valid (V)
- Bits 1-1: Table (T)
- Bits 2-2: User (U)
- Bits 3-5: Access permissions (AP)
- Bits 6-8: Shareability (SH)
- Bits 9-10: Access flag (AF)
- Bits 11-11: Non-secure (NS)
- Bits 12-51: Physical address
- Bits 52-62: Reserved
- Bits 63-63: Execute never (XN)

### ARM64 Page Allocator

**Bitmap-based allocation**:
- Total pages: 524,288 (2GB)
- Page size: 4KB
- Bitmap size: 8192 * 64 bits
- Allocation: O(1) average case
- Deallocation: O(1)

### ARM64 Heap Allocator

**Simple bump allocator**:
- Heap size: 16MB
- Heap start: 0x40980000
- Heap end: 0x41980000
- Alignment: 16 bytes
- Allocation: O(1) average case
- Deallocation: Not implemented (simple allocator)

### ARM64 Cache Management

**Cache operations**:
- `ic iallu`: Invalidate instruction cache
- `dc iallu`: Invalidate data cache
- `dc cvau`: Clean data cache
- `dc civac`: Clean and invalidate data cache
- `dsb ish`: Data synchronization barrier
- `isb`: Instruction synchronization barrier
